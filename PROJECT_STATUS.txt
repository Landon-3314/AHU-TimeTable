Overview
该项目已从“课程展示”演进为“本地优先的课程 + 日程管理应用”，核心能力包括：
- 课程（Course）与单次日程（Event）混合管理
- 教务系统 WebView 抓取导入（含异常结构兼容）
- 多语言（中文/英文）
- Android 自动静音（前台常驻后台服务）
- 课前提醒迁移至后台轮询触发（不再依赖 zonedSchedule）
- 周视图支持“假期中”模式

Current Architecture (Active)
- UI: PageView + ListView + Provider
- Local storage: SharedPreferences
- Import: WebView + JS 注入 DOM 抓取
- Background runtime: flutter_background_service (Android foreground service)
- Notifications: flutter_local_notifications（即时 show；不再注册未来闹钟）

二、关键 Bug 闭环与具体修复代码逻辑
1) 教务导入“识别为空”
- 根因：教务页面常位于 iframe，且旧链路把 JS 诊断字符串直接透传给 UI。
- 修复：
  - `lib/services/schedule_html_extractor.dart`
    - JS 递归遍历 `window.frames`，收集 `table.Wjkc/table.courseTable/.tdHtml` 候选片段。
    - 跨域/抓取异常统一返回 `JS_ERROR:` 前缀字符串。
  - `lib/screens/import_course_page.dart`
    - `runJavaScriptReturningResult` 结果经 `_normalizeJavaScriptResult(...)` 标准化后再处理。
    - `JS_ERROR:` 分支统一抛 `ScheduleParseException('教务系统连接失败，请检查网络或重新登录后重试。')`，
      UI 仅以 SnackBar 友好提示，不再展示原始 JS 报错细节。

2) Event 后台提醒未触发
- 根因：旧方案使用“绝对时刻命中”，系统调度抖动会错过分钟边界。
- 修复：
  - `lib/services/background_runtime_service.dart`
    - 在 `_runEventReminderTick(...)` 中改为“时间窗口 + 去重”策略：
      - 窗口命中：`now >= reminderTime && now < eventEndTime`
      - 去重：`_notifiedEventIds` 按事件 id 去重
      - 过期清理：`_pruneExpiredEventNotifications(...)` 动态移除已失效 id
    - 命中后调用 `NotificationService.showEventReminder(...)`，并加入去重集合。

3) 后台 IPC 断联 / Isolate 启动失败
- 根因：后台入口与监听时序不稳，异常缺少统一兜底。
- 修复：
  - `lib/services/background_service_coordinator.dart`
    - 顶层入口 `@pragma('vm:entry-point') onServiceStart(ServiceInstance service)`。
    - `runZonedGuarded(...)` 包裹隔离区主流程，保留 `debugPrint('Background Error: $error')` 安全网。
    - `AndroidConfiguration.onStart` 明确绑定顶层函数。
  - `lib/services/background_runtime_service.dart`
    - `start(...)` 最先注册 `test_mute/test_unmute/sync_now/stop_service` 监听，再执行后续初始化。

4) DND 权限通过但静音未生效
- 根因：静音执行层缺少紧凑的权限-模式-设置闭环。
- 修复：
  - `lib/services/audio_mode_service.dart`
    - 新增 `executeMute()`：
      - 权限检查（优先动态兼容 `SoundMode.isPermissionGranted`，回退 `PermissionHandler.permissionsGranted`）
      - 仅在当前不是 `vibrate/silent` 时才调用 `SoundMode.setSoundMode(RingerModeStatus.vibrate)`
      - 异常仅 `debugPrint('Mute execution failed: ...')`，避免崩溃。
    - `muteDevice()/muteDeviceSafely()` 统一委托到 `executeMute()`。

Overview
This project is a local-first Flutter course timetable app.
The original implementation explored an absolute-position timetable based on Stack + Positioned, then later pivoted to a more stable flow layout based on PageView + ListView.
The current UI no longer uses absolute-position rendering for the main timetable view.

Current Product Direction
The app currently focuses on:
- Local storage via SharedPreferences
- Global app state via Provider
- Manual course creation and editing
- Day view and week view timetable browsing
- Import framework for future academic-system scraping through WebView

Core Tech Stack
- Flutter
- provider
- shared_preferences
- webview_flutter

Global Architecture
- main.dart
  - Calls WidgetsFlutterBinding.ensureInitialized()
  - Awaits SharedPreferences.getInstance() before runApp()
  - Registers Provider state at app root
  - Uses a custom AppScrollBehavior so Flutter Web supports mouse drag on PageView and scrollables
- MainScaffold
  - BottomNavigationBar with Timetable and Settings
- Providers
  - CourseProvider: course list + currentWeek + persistence
  - SettingsProvider: timetable schedule settings + persistence + generated time slots

Important Historical Context
There were several architecture phases:
1. Initial data layer setup with Course model and CourseProvider
2. App scaffold and persistent settings
3. Absolute-position timetable rendering using Stack + Positioned
4. Short-card overflow fixes and detail sheet interaction
5. Week-based filtering, edit mode, and schedule slot generation
6. Period-based timetable generation with realistic campus break rules
7. Pivot away from absolute-position layout to PageView + ListView
8. Web drag support and WebView import skeleton

Only the latest architecture matters for future work unless we intentionally revert.

Current Data Model
File: lib/models/course.dart

Course fields currently:
- name: String
- location: String
- teacher: String
- weekday: int
- weeks: List<int>
- startPeriod: int
- endPeriod: int
- colorValue: int

Compatibility behavior in Course.fromJson:
- Missing teacher -> ''
- Missing weeks -> [1]
- Missing startPeriod -> 1
- Missing endPeriod -> 2
- Missing weekday -> 1
- Missing location -> ''
- Missing colorValue -> 0xFF7C9AF2
- Missing name -> 'Untitled Course'

This compatibility exists because old local cache formats changed multiple times during development.

generateTimeSlots() currently:
- Builds morning, afternoon, evening periods
- Uses realistic campus rules
- Morning and afternoon: bigBreak after the 2nd class in that session
- Evening: only shortBreak
- Default schedule:
  - classDuration = 45
  - shortBreak = 5
  - bigBreak = 15
  - morningStartTime = 08:00, morningClasses = 5
  - afternoonStartTime = 14:00, afternoonClasses = 5
  - eveningStartTime = 19:00, eveningClasses = 3

Web Import Skeleton
File: lib/screens/import_course_page.dart

Current implementation:
- Uses webview_flutter
- Creates a WebViewController
- Enables unrestricted JavaScript
- Loads placeholder URL: https://www.baidu.com
- Adds JavaScript channel named CourseDataChannel
- Provides a floating action button to inject mock JS
- Injected JS currently:
  - logs to console
  - posts a mock JSON payload back through CourseDataChannel
- The received payload is parsed and imported via CourseProvider.addCourses()
- After success:
  - shows success SnackBar
  - returns to previous page

Important Note About WebView
- The current WebView page is a scaffold for Android/mobile-focused real import later
- Flutter Web may have WebView limitations or cross-origin issues in development
- The import logic is intentionally mock-based for now
- The real future work is to replace the injected mock JS with DOM scraping for the actual academic system

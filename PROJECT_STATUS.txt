目标定位: 多平台本地优先课表，以 Android 系统底层自动化（如自动静音、原生闹钟）为核心体验，同时提供应用内自更新和教务无感化接入的现代化产品。

--------------------------------------------------
一、核心领域模型 (Core Domain Models)
--------------------------------------------------
1. Course (课程)
   - 包含名称、地点、教师、星期、周次数组、开始/结束节次以及颜色配置。
   - 实现了双 ID 机制：`id` 用于唯一追溯，`legacyId` 基于属性生成的 32位 稳定 Hash 码，用于处理早期数据的迁移和防止导入去重错乱。
   - 支持调课追踪 (`rescheduledFromSessionKey`, `rescheduledFromWeek`)。
2. Event (单次日程)
   - 支持在日视图和周视图中与 Course 混合渲染，具备独立的 `dateTime` 和 `enableAlarm` 开关。
3. Semester (学期) & TimeSlot (时间段)
   - `Semester` 控制课表的生命周期、起止日期、周数。
   - `TimeSlot` 抽象了每节课的物理开始/结束时间 (`ClockTime` 24小时制)。
4. UpdateManifest (OTA 清单)
   - 具备 `versionName`, `versionCode`, `baseVersionCode` 等校验维度。
   - 内置 `UpdateAsset` 根据设备的 CPU ABI 进行智能匹配分发，支持主备多个分发镜像 (`mirrorUrls`) 及 `sha256` 完整性校验。

--------------------------------------------------
二、教务账号与全自动化集成
--------------------------------------------------
摒弃了仅依赖手动 WebView 注入的半自动方案，已全面升级为包含静默账号体系的全自动引擎：
1. AcademicCredentialService
   - 使用 `flutter_secure_storage` 安全保存教务系统登录凭据，防止账号泄漏。
2. AcademicAutoLoginService
   - 接管并重放用户的登录会话，在需要拉取新数据或 Cookie 失效时自动重登。
3. AcademicDailyAutoImportService
   - 后台自动执行每日课表快照获取与比对，通过特定的覆盖策略将教务系统新变更的课程无缝覆盖到本地。
4. ScheduleParserService / ScheduleHtmlExtractor
   - 在 WebView 和后台网络请求间共享解析能力，抽离 HTML 表格为结构化的 Course 数据，具备对教务系统脏数据的强大容错性。
5. Exam Overview
   - 新增独立页面 `exam_overview_page.dart` 处理考试时间/地点查询，与常规课表系统并行解耦。

--------------------------------------------------
三、超低功耗的原生调度引擎 (Scheduling & Alarms)
--------------------------------------------------
已完全删除轮询式的 `flutter_background_service`，使用原生推入式队列接管应用级自动化：
1. SchedulePlan / SchedulePlanBuilder
   - App 层级的唯一计算源头：结合课程表、当前日历时间、节次配置、提醒提前时间等，计算出未来 N 天（Android 默认整个学期，其他 14 天）的：
     * `ManagedNotification` (课前提醒 / 日程提醒)
     * `AutoMuteWindow` (自动静音生效区间)
     * `CourseStatusWindow` (用于长截图或桌面小组件的数据源)
2. NativeAlarmService (核心层 - Android)
   - Android 独占功能：利用系统的 `AlarmManager` 设置精确闹钟 (Exact Alarms)。
   - 配合 `PermissionService` 的 DND（勿扰模式）权限，系统会自动在课程开始前推送通知，在上课瞬间将设备静音，并在下课瞬间恢复设备原有铃声状态。
   - 新增 `PersistentCourseReminderManager` 维护安卓通知栏的“常驻今日课表”。
3. LocalNotificationService (回退层)
   - 对于 iOS、macOS 以及未授予高阶权限的 Android，接管 `ManagedNotification` 的基础通知唤醒。不支持自动静音的系统会收到特殊的“请手动静音”的提醒。

--------------------------------------------------
四、存储与自诊断体系
--------------------------------------------------
1. StorageService & Provider
   - 以 `CourseProvider`, `SettingsProvider`, `TimetableViewProvider` 三足鼎立管理 UI 层。
   - 数据全量本地化于 `SharedPreferences`。
2. 灾备与迁移
   - `ExternalDataBackupStore`: 支持将完整的应用状态 (包括配置、课表) 导出为统一格式 JSON 便于跨设备迁移。
   - `CorruptRowDiagnosticStore`: 解析到异常的教务数据或坏数据行时记录诊断日志，方便开发者 (通过 `developer_diagnostics_page.dart`) 排查线上异常。

--------------------------------------------------
五、应用内自更新 (OTA Updater)
--------------------------------------------------
1. 采用 `http` / `cronet_http` 进行高性能、防劫持的下载连接。
2. `UpdateCheckService` 抓取远程 `UpdateManifest`。
3. `UpdateDownloadService` 支持断点、多镜像 (`UpdateMirrorUrls`) 后台下载 APK 产物，并在校验 `sha256` 通过后调用底层包管理服务拉起安装。

--------------------------------------------------
六、当前平台架构能力矩阵
--------------------------------------------------
- Android: 能力释放级别最高。支持教务全自动导入、系统级上课自动静音、AlarmManager 精确触发、长截图滚动捕获 (`LongScreenshotService`)、常驻前台服务、OTA 更新下载。
- iOS / macOS: 业务流程完全可用，支持基础的系统通知提醒。系统权限差异导致的长截图、后台静音和 OTA 安装能力皆安全降级 (no-op)，UI 入口自动隐藏。
- 桌面平台 (Windows / Linux) / Web: 现保留了 Flutter Scaffold 与基本 UI 数据读写，依赖平台专属通道的插件已做环境隔离，不会触发运行时崩溃。

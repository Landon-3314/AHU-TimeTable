Overview
This project has evolved from a simple campus timetable app into a local-first schedule manager that supports:
- Periodic courses
- One-off events
- WebView-based academic-system import
- Local notifications
- Android auto-mute
- Lightweight bilingual UI (Chinese / English)

The main timetable UI is no longer based on absolute positioning. The active architecture is:
- PageView + ListView
- Provider for global state
- SharedPreferences for local persistence

Core Tech Stack
- Flutter
- provider
- shared_preferences
- webview_flutter
- intl
- flutter_local_notifications
- timezone

Important Architecture Direction
- Keep the current PageView/ListView architecture
- Do not revert to the old Stack + Positioned timetable layout
- Import logic is now real DOM parsing logic inside WebView JS injection, not a mock payload
- Notifications and schedule logic are local-first and do not depend on a backend

Import System
File: lib/screens/import_course_page.dart

Current state:
- Real WebView-based import page
- Loads actual AHU WebVPN portal URL
- Enables unrestricted JavaScript
- Registers CourseDataChannel
- Uses injected extractScript to scrape DOM

Current scraper capabilities:
- searches in main document first
- if not found, searches accessible iframes
- supports table.Wjkc and table.courseTable
- parses tdHtml / course-name / lesson-name structure
- avoids duplicate hidden tdHtml copies
- supports multiple courses in one cell
- calculates weekday from td class
- calculates period range from dayPartUnit + rowspan
- returns JSON array matching Course model

Recent parser upgrade:
- fixed irregular week formats like:
  - (1-18周)
  - (1-3,5-18周)
  - (1,3,5周)
  - odd/even week suffixes
- weeks are now expanded into List<int>
- location parsing now strips week/period fragments first, preventing week fragments from leaking into location
- this was specifically added to fix the bug where "1-3" leaked into location on adjusted-course rows

Current import UX behavior:
- success path now only saves imported courses and pops back
- the old import success result popup/snackbar was removed
- debug/error messages from JS are still shown through SnackBar

Known limitation:
- Because this is still DOM scraping against a live academic system, selectors may need adjustment if the university page changes

Notifications
File: lib/services/notification_service.dart

Current features:
- initializes timezone engine
- gets local timezone through MethodChannel('app.timezone')
- supports Android + iOS notification initialization
- requests iOS notification permissions
- requests Android notification permission
- handles exact alarm permission checks on Android

Scheduling features:
- schedule course reminders
- schedule event reminders
- refresh all reminders on data/settings changes
- iOS fallback reminder for auto-mute mode

Current separation:
- Course reminders use settings.reminderAdvanceMinutes
- Event reminders use settings.eventReminderAdvanceMinutes

Android scheduling behavior:
- prefers AndroidScheduleMode.exactAllowWhileIdle
- automatically falls back to AndroidScheduleMode.inexactAllowWhileIdle if exact alarm cannot be used

Current debugging logs in NotificationService:
- course scheduling logs
- event scheduling entry logs
- event skip reason logs
- schedule time / now / diffMinutes logs
- exact scheduling capability logs
- zonedSchedule try-catch with error and stack trace logging

Important diagnosis already confirmed:
- If an event is created too close to its start time, and eventReminderAdvanceMinutes pushes the reminder trigger into the past, scheduling is intentionally skipped
- Example already observed in logs:
  - event at 13:35
  - event reminder = 5 minutes
  - created at 13:34
  - computed reminder time = 13:30
  - skipped because scheduledTime was already in the past

Therefore:
- If testing event reminders, either:
  - set event reminder to 0
  - or create an event far enough in the future

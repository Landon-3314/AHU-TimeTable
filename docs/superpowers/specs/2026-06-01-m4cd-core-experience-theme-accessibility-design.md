# M4c+d 核心体验、主题与无障碍设计

## 背景

`problem.md` 中剩余的 M4 条目跨越课程写入安全、考试浏览、主题、中文模式和无障碍。它们都直接影响普通用户日常使用，适合在一个合并里程碑中交付，但实现时必须拆成可独立验证的模块，避免把业务判断散落在界面代码中。

当前代码已有以下基础：

- 课程 Provider 已能拒绝相同课程身份的重复时段。
- 考试已作为 `Event` 从教务系统导入，并带有 `academic.exam` 来源。
- 应用已有可选品牌调色盘，但根应用始终使用浅色主题。
- 翻译字典和语言字段存在，但主要页面仍有大量中文硬编码，英文模式并不完整。
- 引导层已使用 `BlockSemantics`，课程色块已有初步语义标签。

本里程碑采用模块化合并交付：保留现有 Provider 架构，增加小型纯策略类、专属考试页面、主题模式持久化和必要的组件迁移。

## 目标

- 保存或导入课程前识别不同课程之间的时间冲突，并允许用户确认后继续。
- 保持相同课程重复记录直接拒绝，不因冲突提示放宽重复规则。
- 为课程和日程单项删除提供短时 Undo。
- 增加仅展示教务考试的专属考试视图。
- 为教务考试记录可靠的最近导入时间，旧数据不伪造历史时间。
- 增加 `跟随系统 / 浅色 / 深色` 三档主题模式并持久化。
- 让主要用户界面在深色模式下具有可读表面、边框和文字。
- 正式移除不完整英文模式，只保留中文，并接入中文 Material 控件本地化。
- 修复颜色选项语义标签、引导层中文屏障标签、窄屏和大字体布局问题。

## 非目标

- 不在本里程碑实现课程搜索、教师筛选、图片导出或系统分享。
- 不新增普通日程列表页；普通手动日程继续显示在课表中。
- 不把已结束考试自动删除。
- 不实现 `.arb` 国际化生成流程；仅保留中文后，后续若重启多语言支持再按标准流程实施。
- 不重构拆分 `SettingsProvider`，不统一全部颜色常量职责，不提前实施 M5 架构改造。
- 不承诺真机 TalkBack 顺序、国产 ROM 后台行为或 CI 发布证书连续性已经验证。

## 课程冲突策略

### 纯策略模块

新增 `CourseConflictPolicy`，只负责识别冲突，不直接写存储，也不弹窗。策略返回可用于界面汇总的 `CourseConflict` 列表，每项包含候选课程和与其冲突的已有课程。

两个课程构成时间冲突时必须同时满足：

1. 周几相同。
2. 至少一个教学周重叠。
3. 节次区间重叠。
4. 课程身份不同。

课程身份沿用现有规范化逻辑：课程名、地点和教师去除首尾空白并转为小写后比较。相同身份的重叠时段仍属于重复记录，由现有拒绝规则处理，不进入可确认冲突列表。

### Provider 接口

`CourseProvider` 提供：

- 查询单个或一组候选课程冲突的方法。
- 新增、编辑和调课方法的 `allowConflicts` 参数，默认值为 `false`。
- 导入预检方法：忽略即将被替换的旧 `academic.timetable` 批次，检查新批次与保留课程之间以及新批次内部的冲突。

写入规则：

- 发现重复记录时始终拒绝。
- 发现不同课程冲突且 `allowConflicts == false` 时不写入。
- 用户确认后使用 `allowConflicts: true` 重试，允许写入冲突课程。

### 界面交互

新增、编辑和调课页面先调用 Provider 冲突查询，再决定是否执行写入。发现冲突时显示汇总确认框；确认框列出冲突课程名、星期和节次。用户确认后使用 `allowConflicts: true` 写入，用户取消后不写存储、不刷新提醒。写入方法现有布尔返回值继续只用于判断重复记录等无法写入场景，界面不通过布尔失败反向猜测冲突原因。

教务导入在解析完成后先执行整批预检：

1. 排除即将被替换的旧教务课程批次。
2. 汇总与手动课程及新批次内部的冲突。
3. 只弹出一次确认框。
4. 用户确认后整批替换；用户取消后保持原课程数据不变。

## 单项删除 Undo

### Provider 行为

`CourseProvider` 增加恢复课程和恢复日程的方法。删除方法返回实际删除的记录；恢复方法按 ID 去重后重新写入存储并刷新提醒。

规则：

- 删除课程后立即从界面和存储移除，并显示短时 SnackBar。
- 删除日程后使用相同行为。
- SnackBar 提供“撤销”动作。
- 用户点击撤销时，重新插入原记录、持久化并刷新提醒。
- 若记录已存在，恢复方法不重复插入。
- 清空全部本地数据仍保留强确认，不提供 Undo。

### 界面位置

现有课程和日程详情底部弹层继续承担删除确认。详情弹层函数保留调用方传入的父页面 `BuildContext`：确认后关闭弹层，等待 Provider 删除完成，再使用父页面的 `ScaffoldMessenger` 显示 Undo SnackBar，避免 SnackBar 绑定到已销毁的弹层上下文。

## 专属考试视图

### 页面入口与数据范围

新增 `ExamOverviewPage`，课表工具栏增加考试入口。考试页只展示：

```text
event.importSource == CourseProvider.academicExamImportSource
```

普通手动日程继续留在课表日视图、周视图和假期视图中，不进入考试页。

### 考试卡片

考试按 `dateTime` 升序排列。每张卡片展示：

- 科目名称。
- 日期与时间。
- 倒计时：未来超过一天显示“还有 N 天”，同一天显示“今天”，过去显示“已结束”。
- 地点。
- 座位或备注：当前解析器没有独立座位字段，因此使用现有 `note` 展示；无内容时显示“未设置”。
- 来源：固定展示“教务系统”。
- 最近导入时间：有值时展示本地日期时间，无值时展示“未知”。

点击卡片继续复用现有日程详情弹层。空状态显示“暂无教务考试”，并提供“导入考试”按钮，进入现有教务导入页。

### 数据模型

`Event` 增加可空 `DateTime? importedAt`：

- `toJson` 写入 ISO 8601 字符串或 `null`。
- `fromJson` 对缺失值返回 `null`。
- 字段存在但不可解析时返回 `null`，不让历史可用考试因附加元数据损坏而丢失。
- 每次 `mergeImportedEvents` 使用同一批次时间写入所有新教务考试。
- 手动日程保持 `importedAt == null`。

导入批次 ID 继续保留现有用途，不从 ID 反向推导时间。

## 三档主题模式

### 持久化

新增应用主题模式枚举：

```text
AppThemeMode.system
AppThemeMode.light
AppThemeMode.dark
```

`StorageService` 保存字符串值。读取到未知值时回退为 `system`。

`SettingsProvider` 暴露当前应用主题模式和修改方法。设置页外观区域增加“显示模式”入口，通过单选弹层选择：

- 跟随系统
- 浅色
- 深色

### 根应用

主 `MaterialApp` 配置：

```text
theme: AppTheme.light(...)
darkTheme: AppTheme.dark(...)
themeMode: settingsProvider.themeMode.materialThemeMode
```

初始化加载页和初始化错误页也同时提供亮暗主题，确保系统处于深色时不会短暂闪现浅色页面。

### 主题令牌

保留现有品牌调色盘作为强调色来源。新增 `ThemeExtension<AppThemeTokens>` 承载亮暗两套界面令牌，至少覆盖：

- 页面背景。
- 卡片表面和次级表面。
- 分割线。
- 主、次、弱文本。
- 信息、警告和危险状态表面。

组件优先读取 `Theme.of(context).colorScheme` 和主题扩展令牌。当前直接读取固定浅色 `AppColors.surface`、`surfaceMuted`、`divider`、`textPrimary`、`textSecondary`、`textTertiary` 的主要用户界面逐步迁移；静态品牌调色盘和状态语义色可继续保留。

### 对比度

新增颜色前景策略：对给定背景色在黑色和白色中选择对比度更高者。普通文字目标最低为 WCAG AA `4.5:1`。

该策略用于：

- 品牌按钮与浮动按钮前景。
- 课程颜色色块和主题颜色色块勾选图标。
- 需要在自定义强调色背景上展示文字的徽标。

自定义主题颜色仍只允许从现有预设调色盘选择，因此不新增自由输入校验流程。

## 仅保留中文

本轮正式移除不完整英文模式声明：

- `SettingsProvider.languageCode` 固定返回 `zh`。
- `SettingsProvider.t()` 保留现有调用方式，减少无关重构。
- 删除运行时语言修改能力与语言持久化写入入口；旧版本遗留的语言偏好键可以留在快照中，但运行时忽略。
- 清理设置页和其他可见入口中的语言选择能力。
- `MaterialApp` 配置中文 `supportedLocales` 和标准 `localizationsDelegates`。
- `pubspec.yaml` 增加 `flutter_localizations` SDK 依赖。
- 将当前主要用户路径中可见的英文异常提示改为中文。

现有翻译字典暂不大规模删除，避免制造无关差异；英文值不再作为受支持界面暴露。

## 无障碍与响应式

### 颜色选项

课程颜色和主题颜色为每个色块提供可区分中文语义标签：

```text
颜色 1，青绿
颜色 2，蓝色
```

标签包含序号和颜色名，并继续设置选中状态。选中项必须同时具有视觉勾选，不能只依赖颜色区别。

### 引导层

现有 `BlockSemantics` 保留。引导层屏障标签从固定英文改为中文“功能引导”。步骤卡片将当前步骤和总步骤计数纳入语义播报。

### 窄屏工具栏

课表页顶部在宽度不足时保留周次选择和“今天”，将课程总览、考试、导入和新增课程收敛到更多菜单。常规宽度继续展示高频操作图标，避免牺牲日常效率。

所有图标按钮继续提供中文 tooltip，交互目标不小于 `44dp`。

### 大字体滚轮

滚轮弹层根据当前可用高度约束内容区域。大字体或低高度设备下允许内容滚动，确认和取消操作保持可达，不截断底部按钮。

## 错误处理

- 冲突检测只提示不同课程重叠，不改变相同课程重复拒绝。
- 用户取消冲突确认时不修改数据。
- Undo 已恢复或已存在的 ID 不重复插入。
- 考试 `importedAt` 缺失或损坏时显示“未知”。
- 已结束考试继续展示，便于历史核对。
- 主题模式存储值非法时自动回退“跟随系统”。
- 深色主题不修改课程自定义颜色值，只调整界面表面、边框和前景色。
- 清理英文支持时保留 `t()` 兼容层，避免一次性重写所有文案调用点。

## 测试策略

### 纯策略与 Provider 单测

- 不同课程同星期、同周且节次重叠会报告冲突。
- 相同身份课程继续识别为重复，不进入可确认冲突。
- 不同周、不同星期或不重叠节次不误报。
- 导入预检忽略旧教务批次，检测与手动课程和批次内部冲突。
- `allowConflicts: true` 后冲突课程可保存。
- 删除课程、删除日程和对应 Undo 均持久化并刷新提醒。
- Undo 不重复插入已有 ID。
- 新导入考试写入相同批次导入时间。
- 旧考试缺少或损坏 `importedAt` 仍可读取。
- 未知主题模式存储值回退 `system`。
- 黑白前景策略选择较高对比度颜色，并满足预设调色盘的文字对比度验收。

### Widget 测试

- 新增、编辑和调课课程出现冲突确认框；取消时不保存，确认后保存。
- 教务导入冲突只显示一次整批确认。
- 考试页只展示教务考试，包含倒计时、来源和导入时间。
- 考试空状态可进入导入页。
- 课程和日程删除后显示“撤销”按钮，点击后恢复。
- 设置页展示三档主题模式，修改后根应用正确切换暗色主题。
- 中文 Material 本地化可用。
- 课程色块和主题色块语义标签包含序号、颜色名和选中状态。
- 引导层屏障和步骤语义使用中文。
- `320dp` 宽度下课表工具栏无溢出。
- `200%` 字体下滚轮弹层操作按钮可达且无溢出异常。

## 风险控制

- 冲突计算放入纯策略模块，以单测覆盖边界；Provider 只负责写入门禁。
- 教务考试页复用现有 `Event` 与详情弹层，不引入第二套存储。
- 暗色迁移聚焦主要用户界面，固定浅色常量的替换按组件逐步完成。
- 移除英文支持采用兼容层方式，不同时发动 `.arb` 迁移。
- 所有新增行为按 TDD 顺序实施，先观察失败测试，再写最小修复。
